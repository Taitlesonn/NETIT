Konfiguracja IPv4 i IPv6 na MikroTik

W tej notatce przedstawiamy podstawowe informacje oraz praktyczne przykłady konfiguracji protokołów IPv4 i IPv6 na urządzeniach MikroTik z systemem RouterOS.

1. Wstęp do IPv4 i IPv6

IPv4 to czwórkowy protokół internetowy, który używa adresów 32-bitowych, co pozwala na około 4 miliardy unikalnych adresów. Ze względu na wyczerpywanie się dostępnych adresów, wprowadzono IPv6, który używa adresów 128-bitowych, oferując niemal nieograniczoną liczbę unikalnych adresów.

2. Wymagania wstępne

  • Urządzenie MikroTik z zainstalowanym RouterOS.
  • Aktywne połączenie internetowe (IPv4 lub IPv6).
  • Wersja RouterOS 7.x lub nowsza.

3. Konfiguracja IPv4

Przykład konfiguracji adresu IPv4 na interfejsie Ethernet:

/ip address add address=192.168.1.1/24 interface=ether1

Dodanie domyślnej trasy routingu:

/ip route add gateway=192.168.1.254

Sprawdzenie tabeli routingu:

/ip route print

Ustawienie serwera DHCP dla sieci LAN:

/ip dhcp-server setup

Weryfikacja konfiguracji DHCP:

/ip dhcp-server print

Ustawienie NAT dla udostępnienia internetu:

/ip firewall nat add chain=srcnat out-interface=ether1 action=masquerade

Sprawdzenie reguł NAT:

/ip firewall nat print

Więcej informacji na temat konfiguracji IPv4 można znaleźć w dokumentacji MikroTik: IPv4 and IPv6 Fundamentals.

4. Konfiguracja IPv6

Wersja RouterOS 7.x domyślnie obsługuje IPv6. Aby rozpocząć konfigurację:

/system package enable ipv6
/reboot

Po ponownym uruchomieniu urządzenia, konfiguracja IPv6 jest aktywna.

4.1. Konfiguracja klienta DHCPv6

Jeśli dostawca internetu (ISP) udostępnia adresację IPv6 za pomocą DHCPv6:

/ipv6 dhcp-client add interface=ether1 pool-name=dhcpv6-pool request=prefix

Ustawienie adresu IPv6 na interfejsie LAN:

/ipv6 address add address=::1/64 from-pool=dhcpv6-pool interface=bridge-local

Włączenie reklamowania adresu IPv6:

/ipv6 nd set [ find default=yes ] advertise-dns=yes

Więcej informacji na temat konfiguracji IPv6 można znaleźć w dokumentacji MikroTik: IPv4 and IPv6 Fundamentals.

5. Diagnostyka i testowanie

Sprawdzenie adresów IP na interfejsach:

/ip address print
/ipv6 address print

Wykonanie testu ping dla IPv4:

/ping 8.8.8.8

Wykonanie testu ping dla IPv6:

/ping 2001:4860:4860::8888

Sprawdzenie tabeli ARP dla IPv4:

/ip arp print

Sprawdzenie tabeli sąsiadów dla IPv6:

/ipv6 neighbor print

Więcej informacji na temat diagnostyki można znaleźć w dokumentacji MikroTik: IPv4 and IPv6 Fundamentals.

6. Uwagi końcowe

Konfiguracja IPv6 na urządzeniach MikroTik jest stosunkowo prosta, jednak wymaga odpowiedniego wsparcia ze strony dostawcy internetu. Upewnij się, że Twój ISP udostępnia adresację IPv6 i że Twoje urządzenie MikroTik jest odpowiednio skonfigurowane.
