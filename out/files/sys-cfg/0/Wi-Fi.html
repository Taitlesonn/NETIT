Konfiguracja Wi‑Fi w RouterOS (MikroTik)

Menu /interface wifi dostępne od RouterOS 7.13 służy do zarządzania interfejsami Wi-Fi (802.11ac/ax) – poprzednio nazywane „wifiwave2”. Wymaga aktywnych paczek: wifi‑qcom‑ac (dla ac) lub wifi‑qcom (dla ax+) :contentReference[oaicite:2]{index=2}.

1. Terminologia i profile

  • Profile: aaa, channel, security, datapath, interworking.
  • Konfiguracja: przypisanie profili do interfejsu Wi‑Fi.
  • Priorytet wartości: interfejs → profile → konfiguracja → profile w konfiguracji :contentReference[oaicite:3]{index=3}.

2. Podstawowa konfiguracja AP

/interface wifi
set wifi1 disabled=no configuration.country=PL ssid=MojeAP \
  security.authentication-types=wpa2-psk passphrase="bezpiecznehasło"

Open AP z OWE (Opportunistic Wireless Encryption):

/interface wifi
add master-interface=wifi1 name=wifi1_owe ssid=MojeAP_OWE \
  security.authentication-types=owe \
  security.owe-transition-interface=wifi1 hide-ssid=yes
set wifi1 configuration.country=PL ssid=MojeAP \
  security.authentication-types="" \
  security.owe-transition-interface=wifi1_owe
enable wifi1,wifi1_owe

OWE zapewnia szyfrowanie bez hasła, z opcją przejściową dla urządzeń starszych :contentReference[oaicite:4]{index=4}.

3. Profile bezpieczeństwa i kanały

/interface wifi security
add name=sec-common authentication-types=wpa2-psk,wpa3-psk \
  passphrase="duzeBezpieczneHaslo" wps=disable

/interface wifi configuration
add name=conf-common ssid=MojeAP country=PL security=sec-common

/interface wifi channel
add name=ch-2ghz frequency=2412,2437,2462 width=20mhz
add name=ch-5ghz frequency=5180 width=20/40/80mhz

/interface wifi
set wifi1 channel=ch-5ghz configuration=conf-common disabled=no
set wifi2 channel=ch-2ghz configuration=conf-common disabled=no

4. Lista dostępu (access‑list)

Filtrowanie klientów po sygnale, MAC, czasie itd. Przykład:

/interface wifi access-list
add signal-range=-60..0 ssid-regexp="MojeAP" time=7h-19h action=accept
add ssid-regexp="MojeAP" action=reject
add mac-address=02:00:00:00:00:00 action=reject

Można też przypisać różne hasła dla konkretnych klientów :contentReference[oaicite:5]{index=5}.

5. Diagnoza i skanowanie

  • /interface wifi frequency-scan wifi1 – statystyki obciążenia kanałów.
  • /interface wifi scan – lista AP z sygnałem.
  • Sniffer i spectral scan dostępne dla analizy sygnału i zakłóceń :contentReference[oaicite:6]{index=6}.

6. WPS

/interface wifi wps-client wifi1
/interface wifi wps-push-button wifi1

7. Informacje o radio i tabela klientów

/interface wifi radio print detail
/interface wifi registration-table print
/interface wifi registration-table remove [find where mac-address=XX:XX:XX]

Pozwala to analizować parametry radia, listę klientów oraz ręcznie deautoryzować urządzenia :contentReference[oaicite:7]{index=7}.

8. CAPsMAN – centralne zarządzanie AP

CAPsMAN umożliwia scentralizowane zarządzanie AP:

/interface wifi capsman set enabled=yes ca-certificate=auto
/interface wifi provisioning add action=create-dynamic-enabled ... supported-bands=...
/interface wifi cap set enabled=yes
/interface wifi set wifi1 configuration.manager=capsman-or-local

Szczegółowe opcje dla VLAN, pasm 2.4/5 GHz itd. dostępne w dokumentacji :contentReference[oaicite:8]{index=8}.
